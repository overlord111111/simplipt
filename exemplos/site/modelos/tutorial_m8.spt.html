Modulo 8: Alem dos Limites

Classes, async/await, CLI, build, pacotes, debug, profiling

Introducao

Voce chegou ao ultimo modulo do tutorial. Aqui vamos destravar o que o SimpliPT tem de mais poderoso: orientacao a objetos de verdade, programacao assincrona, ferramentas de build e publicacao de pacotes. Depois disso, voce estra pronto para construir qualquer coisa.

Nao se engane pela posicao — este modulo nao e dificil, e libertador. Cada conceito aqui vai expandir seu cenario de possibilidades.

1. Orientacao a Objetos

SimpliPT tem suporte completo a OOP com sintaxe limpa. Tudo que voce espera de uma linguagem moderna: classes, heranca, polimorfismo.

Declarando uma classe

Use classe para definir, construtor para inicializar e metodo para comportamentos.

simplipt
classe Pessoa
  construtor(nome, idade)
    este.nome = nome
    este.idade = idade
  fim

  metodo apresentar()
    retornar "Ola, sou " + este.nome
  fim
fim

joao = Pessoa("Joao", 30)
falar joao.apresentar()

Repare no este — ele e a referencia para a instancia atual, igual ao self do Python ou this do JavaScript.

Heranca com extende

Uma classe pode herdar todos os metodos e atributos de outra usando extende. Use super para chamar o construtor ou metodo da classe pai.

simplipt
classe Animal
  construtor(nome)
    este.nome = nome
  fim

  metodo falar()
    retornar "..."
  fim
fim

classe Cachorro extende Animal
  construtor(nome, raca)
    super(nome)
    este.raca = raca
  fim

  metodo falar()
    retornar este.nome + " diz: Au Au!"
  fim
fim

rex = Cachorro("Rex", "Pastor")
falar rex.falar()

O metodo falar() foi sobrescrito na classe filha. Isso e polimorfismo na pratica: cada animal pode falar de um jeito diferente, mas o codigo que chama .falar() nao precisa saber qual e o tipo concreto.

Dica: Use heranca quando a relacao for "e um" (Cachorro e um Animal). Para "tem um", prefira composicao (Carro tem um Motor).

2. Async/Await

Programacao assincrona permite que seu codigo execute operacoes demoradas (chamadas de API, leitura de arquivos, consultas em banco) sem travar o programa.

Funcoes assincronas

Use async funcao para definir e esperar para aguardar o resultado.

simplipt
async funcao buscar_dados(url)
  falar "Buscando " + url + "..."
  retornar http.obter(url)
fim

async funcao main()
  dados = esperar buscar_dados("https://api.exemplo.com/dados")
  falar "Recebido: " + dados
fim

esperar main()

Executando em paralelo

Use paralelo para rodar varias tarefas ao mesmo tempo e aguardar todas.

simplipt
async funcao main()
  resultado1, resultado2 = paralelo(
    buscar_dados("https://api.exemplo.com/dados1"),
    buscar_dados("https://api.exemplo.com/dados2")
  )
  falar "Resultado 1: " + resultado1
  falar "Resultado 2: " + resultado2
fim

Enquanto as duas requisicoes rodam em paralelo, o programa nao bloqueia. Isso e essencial para aplicacoes que precisam de performance, como servidores web e ferramentas de linha de comando.

Atencao: esperar so pode ser usado dentro de funcoes marcadas com async. Chamar esperar fora de uma funcao async vai gerar erro.

3. CLI e Build

O SimpliPT vem com ferramentas de linha de comando para construir, depurar e analisar seus programas. Nada de configuracao complexa — um comando resolve.

Build gerando .pyz

Empacote seu script em um unico arquivo .pyz portatil:

bash
simplipt build meu_app.spt

Isso gera meu_app.pyz — um arquivo que roda em qualquer maquina com Python instalado. Ideal para distribuir sem depender do SimpliPT instalado.

Build gerando .exe

Quer um executavel nativo do Windows? So adicionar a flag --exe:

bash
simplipt build --exe meu_app.spt

Pronto. Voce tem um meu_app.exe que roda em qualquer maquina Windows sem nenhuma dependencia.

Modo Debug

Execute passo a passo para entender o fluxo do programa:

bash
simplipt meu_app.spt --debug

O debugger mostra cada linha sendo executada, valores de variaveis e pausa a cada passo. Perfeito para encontrar bugs complicados.

Modo Profile

Descubra onde seu codigo esta gastando tempo:

bash
simplipt meu_app.spt --profile

O profiler exibe uma tabela com o tempo gasto em cada funcao, numero de chamadas e tempo acumulado. Essencial para otimizar performance.

Web Debugger

Para uma experiencia visual completa:

bash
simplipt dashboard

Isso abre um painel web com debugger grafico, visualizador de memoria e console interativo. Tudo rodando no seu navegador.

Dica: Use --debug durante o desenvolvimento e --profile quando quiser otimizar. Sao ferramentas que todo desenvolvedor deveria ter.

4. Pacotes e Ferramentas

O ecossistema do SimpliPT tem ferramentas para gerenciar dependencias, formatar codigo, rodar testes e ate converter para Python puro.

Instalando pacotes

Pacotes sao bibliotecas que estendem o SimpliPT. Instale com:

bash
simplipt modulo install nome_do_pacote

Depois de instalado, e so importar no seu codigo:

simplipt
usar "json"
usar "http"

resposta = http.obter("https://api.github.com")
dados = json.decodificar(resposta.corpo)

Formatador de codigo

Mantenha seu codigo limpo sem esforco:

bash
simplipt formatar arquivo.spt

O formatador ajusta indentacao, espacos e quebras de linha seguindo as convencoes da linguagem. Rode antes de cada commit.

Testes

bash
simplipt test

Roda todos os testes do projeto. Funciona com qualquer framework de testes compatível com Python.

Transpilando para Python

Precisa migrar para Python puro? O SimpliPT pode converter seu codigo automaticamente:

bash
simplipt transpilar arquivo.spt

Gera um arquivo.py equivalente, 100% legivel e pronto para ser publicado no PyPI ou integrado em projetos Python existentes.

Exportando para PyPI

Quer compartilhar sua biblioteca com o mundo? Crie um pacote PyPI com:

bash
simplipt build --pypi minha_biblioteca.spt

Isso gera toda a estrutura necessaria: setup.py, pyproject.toml, pasta do modulo e ate um README basico. Depois e so publicar com pip publish.

Dica: O ecossistema SimpliPT foi desenhado para ser pratico. Voce nao precisa de 5 ferramentas diferentes — um unico comando simplipt faz tudo.

Exercicio 1: Classe Carro

Crie uma classe Carro com atributos marca, modelo, ano e um metodo acelerar() que retornar uma string como "Vrum! O Fusca acelerou!".

Crie duas instancias com marcas diferentes e chame o metodo.

▶ Verificar no Playground

Ver solucao
simplipt
classe Carro
  construtor(marca, modelo, ano)
    este.marca = marca
    este.modelo = modelo
    este.ano = ano
  fim

  metodo acelerar()
    retornar "Vrum! O " + este.modelo + " acelerou!"
  fim
fim

fusca = Carro("VW", "Fusca", 1978)
falar fusca.acelerar()

mustang = Carro("Ford", "Mustang", 2024)
falar mustang.acelerar()

Exercicio 2: Hierarquia de Funcionarios (Desafio)

Crie uma hierarquia de classes: Funcionario → Gerente e Estagiario.

  • Funcionario: construtor com nome e salario_base; metodo calcular_salario() que retornar o salario_base.
  • Gerente: recebe um bonus de 20% sobre o salario_base.
  • Estagiario: recebe metade do salario_base (bolsa-auxilio).

Crie um funcionario de cada tipo e exiba os salarios.

▶ Verificar no Playground

Ver solucao
simplipt
classe Funcionario
  construtor(nome, salario_base)
    este.nome = nome
    este.salario_base = salario_base
  fim

  metodo calcular_salario()
    retornar este.salario_base
  fim
fim

classe Gerente extende Funcionario
  metodo calcular_salario()
    retornar este.salario_base * 1.2
  fim
fim

classe Estagiario extende Funcionario
  metodo calcular_salario()
    retornar este.salario_base * 0.5
  fim
fim

joao = Funcionario("Joao", 3000)
maria = Gerente("Maria", 5000)
ana = Estagiario("Ana", 1500)

falar joao.nome + ": R$ " + joao.calcular_salario()
falar maria.nome + ": R$ " + maria.calcular_salario()
falar ana.nome + ": R$ " + ana.calcular_salario()

Exercicio 3: Busca Assincrona

Use async e esperar para buscar dados de duas APIs diferentes em paralelo e exibir os resultados.

Ver solucao
simplipt
async funcao buscar_dados(url)
  falar "Buscando " + url + "..."
  dados = http.obter(url)
  retornar dados.corpo
fim

async funcao main()
  api1 = "https://api.exemplo.com/dados1"
  api2 = "https://api.exemplo.com/dados2"

  resultado1, resultado2 = paralelo(
    buscar_dados(api1),
    buscar_dados(api2)
  )

  falar "API 1: " + resultado1
  falar "API 2: " + resultado2
  falar "Busca concluida com sucesso!"
fim

esperar main()

Exercicio 4: App CLI com Build (Desafio)

Crie um aplicativo de linha de comando que pergunta o nome do usuario e exibe uma saudacao personalizada. Depois use simplipt build --exe para gerar um executavel.

Ver solucao
simplipt
# saudacao.spt
usar data

funcao main()
  falar "=== Saudacao Personalizada ==="
  nome = ler("Digite seu nome: ")
  hoje = data.hoje()

  falar "Ola, " + nome + "!"
  falar "Hoje eh " + hoje

fim

main()

bash
simplipt build --exe saudacao.spt
