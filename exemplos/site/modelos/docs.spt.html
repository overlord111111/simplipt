Sintaxe

SimpliPT usa indentação e palavras em português. Cada linha é uma instrução. Blocos são delimitados por fim.

# Comentário
x = 10
se x > 5
    falar "maior"
fim

Tipagem Opcional

Declare tipos opcionais com nome: tipo = valor. Tipos disponíveis: texto, inteiro, numero, booleano, nada, lista, dicionario.

nome: texto = "João"
idade: inteiro = 25
altura: numero = 1.75
ativo: booleano = verdadeiro

# Erro de tipo em tempo de execução
x: inteiro = "texto"  # Runtime: incompatível

Variáveis

Tipagem dinâmica. Atribuição com =, atribuição composta com += e -=.

nome = "João"
idade = 25
preco = 19.99
ativo = verdadeiro
nulo = nada
lista = [1, 2, 3]
dict = {chave: valor}

# Atribuição composta
x += 5
y -= 3

# Acesso seguro a dict (retorna nada se chave não existe)
d = {}
falar d.chave  # nada

Condicionais

se, senao, comparadores é, não é, >, <, >=, <=, operadores e, ou, nao.

se idade >= 18 e tem_carteira
    falar "Pode dirigir"
senao se idade >= 16
    falar "Pode aprender"
senao
    falar "Muito novo"
fim

se nome nao é ""
    falar "Ola, {nome}"
fim

Loops

Três tipos de loop: para...ate, para...em, enquanto.

# Para...ate (1 a N inclusive)
para i ate 5
    falar i
fim  # 1 2 3 4 5

# Para...em (iterar coleção)
para item em [10, 20, 30]
    falar item
fim

# Enquanto
x = 0
enquanto x < 5
    x = x + 1
fim

Funções

Funções nomeadas e anônimas. fn é atalho para funcao.

# Função nomeada
fn somar(a, b)
    retornar a + b
fim

# Função anônima (inline)
servidor.rota("GET", "/", fn(req)
    retornar "OK"
fim)

# Função sem retorno retorna nada
fn log(msg)
    falar "[LOG] {msg}"
fim

Classes

Orientação a objetos completa com construtor, métodos, herança e super.

classe Veiculo
    construtor(marca)
        este.marca = marca
    fim
    metodo info()
        retornar "Marca: {este.marca}"
    fim
fim

classe Carro extende Veiculo
    construtor(marca, modelo)
        super(marca)
        este.modelo = modelo
    fim
    metodo info()
        retornar super() + ", Modelo: {este.modelo}"
    fim
fim

meu = novo Carro("Honda", "Civic")
falar meu.info()

Listas

Listas dinâmicas com suporte a indexação e concatenação.

nums = [10, 20, 30]
falar nums[0]      # 10
nums = nums + [40] # Concatena
falar tamanho(nums) # 4
nums[1] = 25       # Modifica

Dicionários

Chaves podem ser texto ou identificador. Acesso via . (ponto) ou [].

pessoa = {nome: "João", idade: 30, "cargo": "Dev"}
falar pessoa.nome     # "João"
falar pessoa.cargo    # "Dev"
falar pessoa.inexiste # nada (seguro)
pessoa.idade = 31

Ternário

Condicional inline. Funciona em qualquer expressão: atribuição, argumento de função, lista, dicionário, list comprehension.

# Sintaxe: <valor se condicao senao fallback>
status = "aprovado" se nota >= 7 senao "reprovado"

# Pode aninhar
r = "A" se nota >= 9 senao "B" se nota >= 7 senao "C"

Template Strings

Dentro de strings com aspas duplas, use {expressão} para interpolar valores.

nome = "Mundo"
falar "Olá, {nome}!"       # "Olá, Mundo!"
falar "2 + 2 = {2 + 2}"  # "2 + 2 = 4"

# Aspas simples não interpolam
falar 'Olá, {nome}!'       # "Olá, {nome}!" (literal)

Pipe Operator

Encadeie chamadas de funções com |> para código mais legível.

# Sem pipe: de dentro pra fora
falar formatar(ordenar(filtrar(lista)))

# Com pipe: esquerda pra direita
lista |> filtrar |> ordenar |> formatar |> falar

List Comprehension

Crie listas transformando e filtrando elementos em uma linha.

# [expressao para var em colecao se filtro]
pares = [x * 2 para x em [1,2,3,4,5] se x % 2 eh 0]
# Resultado: [4, 8]

# Com ternário dentro
itens = [n se n > 0 senao 0 para n em [-1, 0, 1, 2]]
# Resultado: [0, 0, 1, 2]

Corresponder (Match/Case)

Equivalente ao switch/case de outras linguagens.

corresponder status
    caso 200
        falar "OK"
    caso 404
        falar "Não encontrado"
    caso 500
        falar "Erro interno"
    senao
        falar "Desconhecido"
fim

Tratamento de Erros

Capture e trate exceções com tentar...capturar.

tentar
    resultado = 10 / 0
capturar erro
    falar "Erro: {erro}"
fim

Modo Debug

Execute scripts passo a passo, inspecione variáveis e avalie expressões com o depurador embutido.

# Terminal
simplipt --debug script.spt
simplipt --debug -b 10,42 script.spt

# Comandos do debugger:
#  (enter)  Próxima linha
#  s        Passo (step)
#  c        Continuar
#  v        Ver variáveis
#  p 2 + 2  Avaliar expressão
#  l        Ver código fonte
#  q        Sair

Build (Compilar)

Compile scripts SimpliPT para executáveis portáteis sem precisar do Python instalado.

# Criar zipapp portátil (.pyz)
simplipt build script.spt

# Criar executável Windows (.exe) — requer PyInstaller
simplipt build --exe script.spt

# Executar o .pyz
python script.pyz

# O .exe é standalone — não precisa de Python
dist/script.exe

Modo Estrito (--strict)

Checagem estática de tipos sem executar o script. Útil para encontrar erros antes do runtime, em CI/CD ou editores.

# Verificar tipos sem rodar
simplipt --strict script.spt

# Exemplo: vai acusar erro de tipo
# script.spt:
nome: texto = 123      # Erro: declarado como 'texto', valor é 'inteiro'
idade: inteiro = "vinte"  # Erro: incompatível

O analisador infere tipos de literais (texto, número, booleano, lista, dict, nada) e ternários, verificando compatibilidade com as anotações.

Profiling (--profile)

Meça o desempenho do seu script com --profile. Cada função é cronometrada individualmente, mostrando tempo total e contagem de chamadas.

# Rodar com profiling
simplipt --profile script.spt

# Relatório no final:
# Funcao         Chamadas   Tempo (s)    %
# ----------------------------------------
# processar      500        2.3450      78.2
# carregar       1          0.4321      14.4
# salvar         1          0.2219       7.4
# ----------------------------------------
# TOTAL                     2.9990     100.00

JSON Mode (--json)

Execute scripts com saída estruturada em JSON. Útil para integrar SimpliPT com outras ferramentas, CI/CD ou editores.

# Saída JSON
simplipt --json script.spt

# Exemplo de saída:
{"sucesso": verdadeiro, "saida": "Olá!\n", "erro": nulo}

Em caso de erro, retorna tipo_erro (sintaxe/runtime/interno) e a saída parcial capturada até o momento do erro.

Hot-Reload (--hot)

Desenvolvimento de servidores web sem restartar o processo. O SimpliPT monitora o arquivo .spt e recarrega as rotas automaticamente ao salvar.

# Iniciar servidor com hot-reload
simplipt --hot server.spt

# Agora edite server.spt — as rotas recarregam sozinhas
# sem precisar parar e iniciar o servidor

O método servidor.vigiar(caminho) também pode ser chamado diretamente no código. Ele monitora o arquivo em uma thread separada (polling a cada 1s) e recarrega as rotas automaticamente.

Test Runner (simplipt test)

Execute scripts .spt como testes. O runner varre o diretório recursivamente, executa cada arquivo e reporta o resultado.

# Rodar todos os testes no diretório atual
simplipt test

# Rodar testes em diretório específico
simplipt test testes/

# Modo verbose com tempos
simplipt test -v

# Exemplo de saída:
# Rodando 3 teste(s)...
#   PASS  test_math.spt (0.002s)
#   FAIL  test_db.spt
#       Garantia falhou: conexao deve existir
#   PASS  test_api.spt (0.151s)
#
# Resultado: 2/3 passaram, 1 falhou

Dica: use garantir() dentro dos scripts de teste para validações. Qualquer exceção não capturada conta como FAIL.

Scaffolding (simplipt new)

Crie projetos completos com um comando. Gera estrutura de diretórios, CRUD com servidor web + banco SQLite + templates HTML.

# Criar projeto
simplipt new blog

# Estrutura gerada:
#   blog/
#     main.spt          — CRUD + servidor + banco
#     modelos/
#       index.spt.html   — template com lista + formulário
#     estatico/
#       style.css        — tema dark profissional

# Rodar
# cd blog && simplipt main.spt

O template gerado inclui: servidor web na porta 3000, SQLite, API REST (GET/POST), interface HTML com formulário e lista, CSS dark moderno.

Async/Await

Funções assíncronas com async funcao e esperar. Usa asyncio do Python por baixo para I/O concorrente.

# Declarar função assíncrona
async funcao buscar_dados(id)
    # Simula I/O demorado
    pausa 0.1
    retornar {id: id, nome: "Item {id}"}
fim

# Chamar com esperar
async funcao main()
    resultado = esperar buscar_dados(42)
    falar resultado.nome
fim

# Executar
esperar main()

# Iniciar debugger web

Módulos Externos

Instale pacotes Python e use como módulos SimpliPT. O sistema baixa o pacote via pip e cria um proxy automático.

# Instalar pacote Python como módulo SimpliPT
simplipt modulo install requests

# Usar no código
usar requests
resposta = requests.get("https://api.exemplo.com")
falar resposta.status_code

# Listar módulos instalados
simplipt modulo listar

# Instalar com nome diferente
simplipt modulo install beautifulsoup4 bs4

HTMX — Frontend Interativo

O template engine do SimpliPT inclui helpers htmx.* que geram HTML com atributos htmx — frontend dinâmico sem JavaScript.

Basta incluir o script htmx via CDN no cabeçalho (já incluso no template padrão) e usar os helpers nos seus .spt.html.

{% raw %}{% incluir "cabecalho.spt.html" %}{% endraw %}

<!-- Botão que faz GET e coloca resultado em #alvo -->
{{ htmx.botao("Carregar", "/api/dados", "get", "#alvo") }}

<!-- Link que faz POST -->
{{ htmx.link("Salvar", "/api/salvar", "post", "#msg") }}

<!-- Botão de deletar com confirmação -->
{{ htmx.deletar("/api/item/1", "#lista", "Tem certeza?") }}

<!-- Input que valida ao sair do campo (blur) -->
{{ htmx.input_texto("email", "/api/validar", "blur", "#erro") }}

<!-- Área que atualiza sozinha a cada 5s -->
{{ htmx.mostrar("/api/status", "5") }}

<!-- Formulário que envia via POST -->
{{ htmx.formulario("/api/cadastrar") }}
    <input name="nome">
    <button>Enviar</button>
</form>

Helpers disponíveis

HelperDescrição
botao(texto, url, metodo?, alvo?, classe?, extras?)Gera <button> com atributo hx-{metodo}
link(texto, url, metodo?, alvo?, classe?, extras?)Gera <a> com atributo hx-{metodo}
input_texto(nome, url, gatilho?, alvo?, classe?)Gera <input> com hx-post e hx-trigger
formulario(url, metodo?, alvo?, enctype?, classe?)Gera <form> com hx-{metodo}
mostrar(url, intervalo_segundos?, metodo?, alvo?, classe?)Gera <div> com polling automático
enviar(url, metodo?, alvo?, gatilho?)Botão "Enviar" para submissão
deletar(url, alvo?, confirmacao?, classe?)Botão deletar com confirmação opcional
carregar(url, alvo, modo?, classe?)Carrega HTML no alvo ao clicar

Parâmetros com ? são opcionais. Os helpers geram HTML puro — você pode inspecionar os atributos no navegador ou complementar com outros atributos estáticos.

Servidor LSP

Integração com editores via Language Server Protocol. Fornece diagnóstico em tempo real, autocomplete, informações ao passar o mouse e lista de símbolos do documento.

# Iniciar servidor LSP (stdin/stdout)
simplipt lsp

# Use com VS Code, Neovim, Emacs, ou qualquer editor LSP

Funcionalidades do LSP:

  • Diagnóstico — erros de sintaxe e léxico em tempo real ao digitar
  • Completion — palavras-chave (se, para, funcao, etc.) e módulos stdlib
  • Hover — descrição de palavras-chave e definições do documento
  • Document Symbols — lista de funções, variáveis e classes definidas

Extensão VS Code

Extensão oficial do SimpliPT para VS Code na pasta vscode-simplipt/.

# Instalar localmente:
# 1. Abrir VS Code
# 2. Extensões (Ctrl+Shift+X) → ... → Install from VSIX
# 3. Ou copiar vscode-simplipt/ para ~/.vscode/extensions/

Recursos da extensão:

  • Syntax Highlight — 7 categorias de tokens com cores distintas
  • 20 Snippets — se, para, funcao, rota, servidor, ternário, classe, etc.
  • Hover — descrição de palavras-chave ao passar o mouse
  • Comandos — Run, Dashboard, Format, Start LSP pelo Command Palette
  • Language Config — auto-fechamento de brackets, folding markers

Garantir (Assert)

Verifique condições com garantir(condicao, "mensagem"). Lança ErroRuntime se a condição for falsa. Ideal para pré-condições, pós-condições e testes.

# Sintaxe básica
x = 10
garantir(x > 0, "x deve ser positivo")

# Mensagem opcional (mostra a expressão se omitida)
garantir(x < 100)

# Útil em funções
fn dividir(a, b)
    garantir(b != 0, "divisor nao pode ser zero")
    retornar a / b
fim

banco.modelo() — ORM

Crie um ORM automático a partir de um schema. O banco cria a tabela e retorna um objeto com métodos CRUD.

usar banco

banco.conectar("app.db")

# Definir schema → tabela + ORM automático
Usuario = banco.modelo("usuarios", {nome: "texto", idade: "inteiro"})

# CRUD sem escrever SQL
id = Usuario.criar(nome: "João", idade: 30)
u = Usuario.buscar(id)
Usuario.atualizar(id, idade: 31)
todos = Usuario.todos()
Usuario.deletar(id)
total = Usuario.contar()

Tipos suportados no schema: texto, inteiro, real, booleano, binario. O campo id é criado automaticamente.
