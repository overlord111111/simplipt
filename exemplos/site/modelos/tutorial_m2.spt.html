Modulo 2: Controle de Fluxo

Aprenda a controlar o fluxo do seu programa: decisoes com se/senao, repeticoes com para/enquanto e correspondencia de padroes com corresponder/caso.

Introducao

Nem todo programa executa linhas do inicio ao fim sem desvios. Na maioria das vezes voce precisa tomar decisoes, repetir blocos de codigo ou escolher entre varios caminhos. Essas estruturas sao chamadas de controle de fluxo e sao a base de qualquer linguagem de programacao que se preze.

No SimpliPT voce tem quatro ferramentas principais para controlar o fluxo: se, para, enquanto e corresponder. Vamos conhecer cada uma delas.

Aula 1: se / senao / senao se — Decisoes

O se permite executar um bloco de codigo apenas se uma condicao for verdadeira. E o senao executa quando a condicao for falsa. Voce tambem pode encadear varias condicoes com senao se.

SimpliPT
se idade >= 18
    falar "Maior de idade"
senao
    falar "Menor de idade"
fim

O SimpliPT usa 2 espacos de indentacao dentro dos blocos. Sempre mantenha a indentacao consistente!

Dica: Use 2 espacos para indentar blocos dentro de se, para, enquanto e corresponder.

Exemplo: Numero par ou impar

SimpliPT
numero = 7
se numero % 2 eh 0
    falar "O numero {numero} eh par"
senao
    falar "O numero {numero} eh impar"
fim

Exemplo: Nota para conceito (senao se)

SimpliPT
nota = 85
se nota >= 90
    conceito = "A"
senao se nota >= 80
    conceito = "B"
senao se nota >= 70
    conceito = "C"
senao se nota >= 60
    conceito = "D"
senao
    conceito = "F"
fim
falar "Conceito: {conceito}"

Aninhamento de condicionais

Voce pode colocar um se dentro de outro. Isso se chama aninhamento e serve para validar multiplas condicoes em sequencia.

SimpliPT
se usuario.logado
    se usuario.admin
        falar "Bem-vindo, admin!"
    senao
        falar "Bem-vindo, usuario!"
    fim
senao
    falar "Faca login para continuar"
fim

Aula 2: para ate e para em — Repeticao

O para repete um bloco de codigo um numero determinado de vezes. Existem duas formas:

  • para var ate N: conta de 1 ate N
  • para var em colecao: itera sobre cada elemento de uma lista ou dicionario

Exemplo: Tabuada do 5

SimpliPT
num = 5
para i ate 10
    falar "{num} x {i} = {num * i}"
fim

Exemplo: Iterando sobre uma lista

SimpliPT
frutas = ["maca", "banana", "laranja"]
para fruta em frutas
    falar "Eu gosto de {fruta}"
fim

Exemplo: Somar de 1 a 100

SimpliPT
soma = 0
para i ate 100
    soma = soma + i
fim
falar "Soma total: {soma}"

Aula 3: enquanto — Loop Condicional

O enquanto repete um bloco enquanto uma condicao for verdadeira. Diferente do para, voce nao sabe quantas repeticoes vao acontecer — o loop so para quando a condicao se torna falsa.

Atencao: Se a condicao nunca se tornar falsa, o loop executa para sempre — isso eh um loop infinito! Sempre tenha uma saida dentro do bloco (uma variavel que muda, um parar, etc.).

Exemplo: Contagem regressiva

SimpliPT
contador = 10
enquanto contador > 0
    falar contador
    contador = contador - 1
fim
falar "Fogo!"

Exemplo: Menu interativo

SimpliPT
opcao = ""
enquanto opcao nao eh "3"
    falar "1 - Ver saldo"
    falar "2 - Depositar"
    falar "3 - Sair"
    opcao = ler("Escolha: ")
    se opcao eh "1"
        falar "Seu saldo eh R$ 100,00"
    senao se opcao eh "2"
        falar "Deposito realizado!"
    fim
fim

Aula 4: corresponder / caso — Match/Case

O corresponder compara um valor com varios padroes e executa o bloco do primeiro caso que corresponder. Eh como um se multiplo, mas mais limpo e legivel.

SimpliPT
dia = 3
corresponder dia
    caso 1
        falar "Domingo"
    caso 2
        falar "Segunda"
    caso 3
        falar "Terca"
    caso 4
        falar "Quarta"
    caso 5
        falar "Quinta"
    caso 6
        falar "Sexta"
    caso 7
        falar "Sabado"
fim

Exemplo: Classificar notas com se/senao se

Nota: corresponder compara valores exatos, nao expressoes. Para intervalos e comparacoes, use se/senao se.

SimpliPT
nota = 85
se nota >= 90
    falar "Excelente!"
senao se nota >= 70
    falar "Bom"
senao se nota >= 50
    falar "Regular"
senao
    falar "Precisa melhorar"
fim
