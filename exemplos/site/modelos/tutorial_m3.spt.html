Modulo 3: Funcoes

Aprenda a criar funcoes para organizar, reutilizar e dar nome a blocos de codigo. Descubra parametros, retorno, funcoes anonimas e o operador pipe.

Introducao

Conforme seus programas crescem, voce percebe que precisa repetir a mesma logica varias vezes. Funcoes resolvem isso: sao como receitas que voce escreve uma vez e usa quantas vezes quiser. Elas organizam o codigo, evitam repeticao e tornam tudo mais facil de ler e dar manutencao.

No SimpliPT, toda funcao comeca com fn e termina com fim. Vamos aprender do basico ate tecnicas avancadas como pipe operator.

Aula 1: Criando Funcoes

Para criar uma funcao use a palavra-chave fn, de um nome para ela, liste os parametros (se houver) entre parenteses e escreva o corpo entre fn e fim.

SimpliPT
fn saudacao(nome)
    falar "Ola, {nome}!"
fim

# Chamando a funcao
saudacao("Over")

Exemplo: Funcao de boas-vindas

SimpliPT
fn boas_vindas(usuario, hora)
    se hora < 12
        falar "Bom dia, {usuario}!"
    senao se hora < 18
        falar "Boa tarde, {usuario}!"
    senao
        falar "Boa noite, {usuario}!"
    fim
fim

boas_vindas("Jarbs", 14)

Note que as funcoes precisam ser definidas antes de serem chamadas (ou o interpretador precisa encontra-las no mesmo arquivo).

Aula 2: Parametros e Retorno

Parametros sao as entradas da funcao — valores que voce passa para ela trabalhar. O retorno eh a saida — o valor que a funcao devolve para quem a chamou.

Use retornar valor para devolver um resultado.

SimpliPT
fn soma(a, b)
    retornar a + b
fim

resultado = soma(10, 5)
falar resultado  # 15

Exemplo: Quadrado de um numero

SimpliPT
fn quadrado(x)
    retornar x * x
fim

falar quadrado(7)  # 49

Exemplo: Verificar se eh par

SimpliPT
fn eh_par(n)
    retornar n % 2 eh 0
fim

falar eh_par(10)  # Verdadeiro
falar eh_par(7)   # Falso

Dica: Se uma funcao nao tiver retornar, ela devolve Nada (o equivalente a null de outras linguagens).

Exemplo: Fatorial (iterativo)

SimpliPT
fn fatorial(n)
    resultado = 1
    para i ate n
        resultado = resultado * i
    fim
    retornar resultado
fim

falar fatorial(5)  # 120

Aula 3: Funcao Anonima e Escopo

Uma funcao anonima eh uma funcao sem nome. Voce pode guarda-la em uma variavel, passar como argumento para outra funcao ou usa-la rapidamente sem precisar declarar um nome.

A sintaxe usa funcao(params) ... fim em vez de fn nome.

SimpliPT
# Guardando funcao anonima em uma variavel
dobrar = funcao(x)
    retornar x * 2
fim

falar dobrar(5)  # 10

Escopo de variaveis

Variaveis criadas dentro da funcao nao vazam para fora. Elas existem apenas durante a execucao da funcao e sao destruidas no final. Isso se chama escopo local.

SimpliPT
x = 10  # variavel global

fn teste()
    x = 99  # variavel local - nao altera a global
    falar "Dentro: {x}"
fim

teste()       # Dentro: 99
falar x       # 10 (global nao foi alterada)

Atencao: Cuidado para nao confundir variaveis locais com globais. Se precisar alterar uma variavel global dentro de uma funcao, use a palavra-chave global (se disponivel na sua versao do SimpliPT).

Aula 4: Pipe Operator | >

O operador pipe |> pega o resultado da expressao da esquerda e passa como argumento para a funcao da direita. Ele permite encadeamento elegante de operacoes.

valor |> funcao eh equivalente a funcao(valor).

SimpliPT
fn quadrado(x)
    retornar x * x
fim

fn text(valor)
    retornar "Resultado: {valor}"
fim

# Sem pipe:
falar text(quadrado(5))

# Com pipe (mais legivel):
falar 5 |> quadrado |> text

Exemplo: Ordenar e pegar o primeiro

SimpliPT
nomes = ["Zara", "Ana", "Beto", "Caio"]
# Pipe encadeado: ordena a lista e pega o primeiro elemento
primeiro = nomes |> ordenar |> primeiro
falar primeiro  # Ana

Exemplo: Encadeamento com funcoes proprias

SimpliPT
fn dobrar(n)
    retornar n * 2
fim

fn somar_um(n)
    retornar n + 1
fim

# 5 -> dobrar -> 10 -> somar_um -> 11
resultado = 5 |> dobrar |> somar_um
falar resultado  # 11

Dica: O pipe funciona com qualquer funcao que receba um unico argumento. Para funcoes com multiplos parametros, use funcao anonima como intermediario: valor |> funcao(x) soma(x, 2) fim.
