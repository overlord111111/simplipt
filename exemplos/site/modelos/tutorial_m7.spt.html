Modulo 7: Banco de Dados

Armazene e consulte dados com SQLite e o ORM embutido do SimpliPT.

Introdução

O que é um banco de dados? É onde você guarda informações de forma persistente — diferente de variáveis em memória que desaparecem quando o programa fecha. O SimpliPT usa SQLite embutido: zero configuração, zero instalação, o banco vive em um único arquivo .db.

O modulo banco oferece um ORM (Object-Relational Mapping) próprio. Você define modelos que representam tabelas, e o ORM traduz chamadas de função em SQL automaticamente. Nada de escrever SELECT * FROM na mao.

Conectando e Definindo Modelos

Antes de tudo, conecte ao banco com banco.conectar(). Depois use banco.modelo() para definir a estrutura.

simplipt
usar banco

# Conecta (cria o arquivo se não existir)
banco.conectar("meu_banco.db")

# Define o modelo "Tarefa" na tabela "tarefas"
Tarefa = banco.modelo("tarefas", {
    titulo: "texto",
    feita: "booleano"
})

O banco e criado no arquivo meu_banco.db. A tabela tarefas e criada automaticamente com as colunas que você definiu, mais um id auto-incrementado.

Criar e Listar (CREATE / READ)

Com o modelo definido, use criar() para inserir registros, todos() para listar e buscar() para encontrar por ID.

simplipt
# Criar registros (argumentos nomeados)
Tarefa.criar(titulo="Estudar SimpliPT", feita=falso)
Tarefa.criar(titulo="Fazer exercicios", feita=falso)
Tarefa.criar(titulo="Publicar pacote", feita=falso)

# Listar todos os registros
todas = Tarefa.todos()
para t em todas
    falar t.id + ": " + t.titulo
fim
# 1: Estudar SimpliPT
# 2: Fazer exercicios
# 3: Publicar pacote

# Buscar por ID
uma = Tarefa.buscar(1)
falar uma.titulo  # "Estudar SimpliPT"

todos() retorna uma lista de dicionários. Cada dicionário tem as chaves do modelo mais o id. buscar() retorna um único dicionário ou nada se o ID não existir.

Filtrando com WHERE

Use onde() para buscar registros que atendem uma condição. O ? e um placeholder para o valor — isso evita ataques de injeção SQL.

simplipt
# Buscar tarefas pendentes (feita = 0/falso)
pendentes = Tarefa.onde("feita = ?", 0)
falar "Pendentes: " + tamanho(pendentes)

Atualizar e Deletar (UPDATE / DELETE)

Para modificar um registro, chame atualizar(id, chave=valor, ...). Para remover, chame deletar(id).

simplipt
# Marcar tarefa 1 como concluida
Tarefa.atualizar(1, feita=verdadeiro)

# Verificar
tarefa = Tarefa.buscar(1)
falar tarefa.feita  # verdadeiro

# Deletar tarefa 2
Tarefa.deletar(2)
Atenção: deletar() remove permanentemente. Não tem "lixeira" — o registro vai embora de vez.

Consultas Avancadas

O ORM do SimpliPT suporta filtros complexos, contagem e ordenacao.

simplipt
# Contar registros
total = Tarefa.contar()
falar "Total de tarefas: " + total

# Contar com filtro
concluidas = Tarefa.contar("feita = ?", 1)
falar "Concluidas: " + concluidas

# Busca com LIKE (parte do texto)
estudo = Tarefa.onde("titulo LIKE ?", "%SimpliPT%")

# Ordenar resultados
recentes = Tarefa.todos("id DESC")

# Multiplos filtros
antigas = Tarefa.onde("id < ? AND feita = ?", 10, 0)

O placeholder ? funciona para qualquer valor. O ORM escapa automaticamente, então você não precisa se preocupar com injeção SQL.

Mini-projeto: API de Tarefas persistente

Combinando banco + servidor, você tem uma API REST completa em poucas linhas:

simplipt
usar servidor
usar banco

banco.conectar("tarefas.db")

Tarefa = banco.modelo("tarefas", {
    titulo: "texto",
    feita: "booleano"
})

servidor.rota("GET", "/tarefas", fn(req)
    retornar Tarefa.todos()
fim)

servidor.rota("POST", "/tarefas", fn(req)
    Tarefa.criar(titulo=req.corpo.titulo, feita=falso)
    retornar { sucesso: verdadeiro }
fim)

servidor.rota("PUT", "/tarefas/:id", fn(req)
    Tarefa.atualizar(numero(req.params.id), feita=verdadeiro)
    retornar { sucesso: verdadeiro }
fim)

servidor.rota("DELETE", "/tarefas/:id", fn(req)
    Tarefa.deletar(numero(req.params.id))
    retornar { sucesso: verdadeiro }
fim)

servidor.iniciar(3000)

Isso e um CRUD completo com banco SQLite, servidor HTTP e JSON, tudo embutido no SimpliPT. Rode e teste com curl ou pelo navegador.

3. Desafio: CRUD completo com servidor

Crie uma API REST para gerenciar produtos com as quatro operações usando banco.modelo() e o modulo servidor:

  • GET /produtos — listar todos
  • POST /produtos — criar novo (corpo JSON com nome e preco)
  • PUT /produtos/:id — atualizar preco
  • DELETE /produtos/:id — deletar
Ver solucao
simplipt
usar servidor
usar banco

banco.conectar("produtos.db")

Produto = banco.modelo("produtos", {
    nome: "texto",
    preco: "numero"
})

servidor.rota("GET", "/produtos", fn(req)
    retornar Produto.todos()
fim)

servidor.rota("POST", "/produtos", fn(req)
    Produto.criar(nome=req.corpo.nome, preco=req.corpo.preco)
    retornar { sucesso: verdadeiro }
fim)

servidor.rota("PUT", "/produtos/:id", fn(req)
    Produto.atualizar(numero(req.params.id), preco=req.corpo.preco)
    retornar { sucesso: verdadeiro }
fim)

servidor.rota("DELETE", "/produtos/:id", fn(req)
    Produto.deletar(numero(req.params.id))
    retornar { sucesso: verdadeiro }
fim)

servidor.iniciar(3000)
